## Critical Issues

### [AR-02] Stored XSS — Admin Dashboard
- **Agent:** Architecture
- **File:** `server.ts:604–611`
- **Issue:** User emails interpolated directly into server-rendered HTML with no escaping. A user registering with `<script>alert(1)</script>@x.com` executes script in the admin session — Basic Auth provides zero protection against this.
- **Fix:**
  ```ts
  const esc = (s: string) => s.replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;');
  ```
  Apply `esc()` to every `r.email`, `r.plan`, `r.created_at` value in the admin table template.
- **Auto-fixable:** No

### [AR-02] Open Email Relay — `/api/send-brief`
- **Agent:** Architecture
- **File:** `server.ts:499–547`
- **Issue:** No authentication check. Any unauthenticated caller can POST any email address + HTML content and send Resend emails from your domain. Burns Resend quota, risks domain spam-listing.
- **Fix:** Add at top of handler:
  ```ts
  const user = getUserBySession(req.cookies?.session_token);
  if (!user) return res.status(401).json({ error: 'Not authenticated' });
  ```
- **Auto-fixable:** No

### [AR-03] No React ErrorBoundary
- **Agent:** Architecture (compound with CQ-03)
- **File:** `src/main.tsx` (missing entirely)
- **Issue:** Any uncaught render error (e.g., calling `.map()` on a non-array from a malformed API response) crashes the entire app to a blank screen with no recovery path.
- **Fix:** Wrap `<App />` in an ErrorBoundary in main.tsx:
  ```tsx
  import { ErrorBoundary } from 'react-error-boundary';
  // fallback: <div>Something went wrong — please refresh.</div>
  ```
- **Auto-fixable:** No

### [CQ-03] JSON.parse Without Shape Validation — generateBrief
- **Agent:** Code Quality
- **File:** `src/lib/generateBrief.ts:231`, `:347`
- **Issue:** Gemini responses parsed with `JSON.parse(response.text || "{}")` and cast as `any` with no structural validation. Empty `{}` fallback silently passes the field-presence check and produces a blank brief that crashes render (compound with AR-03).
- **Fix:** After the field-presence loop, validate types: assert `companySnapshot.overview` is a non-empty string, all array fields are non-empty arrays. Extract a `parseAndValidateGeminiResponse<T>()` utility for reuse.
- **Auto-fixable:** No

### [CQ-03] JSON.parse Without try/catch — Server Routes
- **Agent:** Code Quality
- **File:** `server.ts:554`, `:580`
- **Issue:** `JSON.parse(brief.brief_data)` in the public and authenticated brief GET endpoints with no try/catch. Corrupted DB row throws unhandled exception, leaks stack trace to client.
- **Fix:**
  ```ts
  try {
    brief.brief_data = JSON.parse(brief.brief_data);
  } catch {
    return res.status(500).json({ error: 'Brief data corrupted' });
  }
  ```
- **Auto-fixable:** No

### [FD-05] Unlabeled Form Inputs (7 instances)
- **Agent:** Frontend/Design
- **Files:** `src/App.tsx:404,417,430,453` | `src/components/PrepBrief.tsx:469` | `src/components/AuthPanel.tsx:92` | `src/components/SignInGate.tsx:76`
- **Issue:** All form inputs use placeholder as the only label. Screen readers cannot associate labels with fields.
- **Fix:** For each input, add `id` + matching `htmlFor` on the label. For email-only inputs with no visible label, use `className="sr-only"`:
  ```tsx
  <label htmlFor="company-name" className="...">Company name</label>
  <input id="company-name" ... />
  ```
  For MCQ pill groups: wrap in `<fieldset><legend>...</legend></fieldset>`.
- **Auto-fixable:** No

### [FD-03] focus:outline-none Without focus-visible Scoping (5 instances)
- **Agent:** Frontend/Design
- **Files:** `src/App.tsx:412,425,438` | `src/components/AuthPanel.tsx:98` | `src/components/SignInGate.tsx:82` | `src/components/PrepBrief.tsx:476`
- **Issue:** `focus:outline-none focus:ring-2` fires on both mouse clicks and keyboard — the ring flashes on click and doesn't guarantee UA outline is replaced in all browsers.
- **Fix:** Replace all instances of `focus:outline-none focus:ring-2 focus:ring-zinc-*` with `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-zinc-*`.
- **Auto-fixable:** No

### [FD-18] Dynamic UI Without aria-live (5 instances)

### [AR-02] Prompt Injection — Unvalidated Enum Fields in generate-brief
- **Agent:** Architecture
- **File:** `server.ts:386–416`
- **Issue:** `round`, `familiarity`, `timeToPrep`, `biggestGap` are interpolated directly into the Gemini system prompt with no enum validation. An attacker can inject arbitrary text into the prompt via these fields.
- **Fix:** Validate all enum fields server-side before passing to `generateBrief()`. Use a strict allowlist:
  ```ts
  const VALID_ROUNDS = ['phone_screen', 'technical', 'behavioral', 'final', 'onsite'];
  if (!VALID_ROUNDS.includes(round)) return res.status(400).json({ error: 'Invalid round' });
  ```
- **Auto-fixable:** No

### [CQ-06] TypeScript Strict Mode Disabled
- **Agent:** Code Quality (compound with AR-03)
- **File:** `tsconfig.json`
- **Issue:** `"strict": true` absent. `strictNullChecks`, `noImplicitAny`, and 5 other safety checks are off. Root cause of why `as any` casts throughout db.ts and generateBrief.ts are invisible to the compiler.
- **Fix:** Add `"strict": true` to `compilerOptions`. Treat resulting errors as a bug queue — prioritize nullability on API response types that flow into render.
- **Auto-fixable:** No

## Learning Loop Additions (Patterns seen 3+ times — do NOT modify lessons.md directly)

1. **`import React` in React 17+ / React 19 projects** — fires in every AI-generated component file when the project uses `react-jsx` transform. ESLint rule: `"react/react-in-jsx-scope": "off"`. Auto-fixable once ESLint is configured.

2. **`focus:outline-none` without `focus-visible:` scoping** — fires on every hand-rolled input in AI-generated React projects. All form input ring/outline patterns should default to `focus-visible:` scope.

3. **Decorative inline SVGs without `aria-hidden="true"`** — fires on every inline SVG in every component. All decorative SVGs need `aria-hidden="true"` as a baseline.

4. **`JSON.parse()` on LLM responses with no shape validation** — fires on every new AI API call added to this codebase. Establish `parseAndValidateGeminiResponse<T>()` utility to enforce once.

5. **Missing `type="button"` on non-submit buttons in or near forms** — fragile default that AI consistently omits.
